# Work 3 — Apache Spark (PySpark)

Processing large datasets using **Apache Spark** with the PySpark API. Spark's in-memory distributed computation provides significantly faster processing than MapReduce for iterative and multi-step queries. The work covers the same two problem domains as Works 1 and 2: a public movie ratings dataset and the electric vehicle parts business dataset.

---

## Technology Stack

| Tool | Version | Role |
|------|---------|------|
| Python | 3.x | Implementation language |
| Apache Spark | 3.x | Distributed in-memory processing engine |
| PySpark | 3.x | Python API for Apache Spark |
| HDFS | — | Distributed file system for input data |

## Business Context

The business dataset represents an **electric vehicle spare parts importer and distributor** operating in Colombia. The company:

- Sources products from **multiple international suppliers**, each with their own catalog and pricing.
- Sells exclusively to **businesses, specialized workshops, and authorized distributors**.
- Manages a network of **50 warehouses** distributed across Colombian cities.
- Tracks all purchases and sales through an **electronic invoicing system**.

The TSV input files are generated by `Trabajo_1/Tablas/generar_datos.py`. They are **not included** in this repository — generate them locally and upload to HDFS before running any script (see [How to Run](#how-to-run) below).

---

## Data Model

All business data is stored as **tab-separated (TSV) files**.

| Table | Rows | Description |
|-------|------|-------------|
| `REPUESTO.tsv` | 100,000 | Spare parts catalog — id, SKU, name, category, brand |
| `INVENTARIO.tsv` | ~50,000 | Stock per warehouse/part — available, reserved, reorder point, avg cost |
| `DETALLE_VENTA.tsv` | ~50,000 | Sales invoice line items — quantities, unit prices |
| `ORDEN_COMPRA.tsv` | 5,000 | Purchase orders — order id, supplier, date, status |

**Part categories:** Motor, Transmision, Frenos, Suspension, Electrico, Carroceria, Refrigeracion, Direccion, Neumaticos, Filtros

**Purchase order statuses:** Pendiente, Confirmada, En_transito, Recibida, Cancelada

## MovieLens Dataset

`u.data` and `u.item` are from the [MovieLens 100K](https://grouplens.org/datasets/movielens/100k/) dataset.

| File | Separator | Schema |
|------|-----------|--------|
| `u.data` | Space | user_id, movie_id, rating, timestamp |
| `u.item` | Pipe (`\|`) | movie_id, title, release_date, video_release, imdb_link, ... |

`u.data` is read from **HDFS** (`hdfs://namenode:9000/user/root/input/u.data`).  
`u.item` is read from the **local filesystem** of the Spark driver node (`/tmp/u.item`).

These files are **not included** in the repository. Download the dataset and place them in the appropriate locations before running `Peliculas.py`.

## Scripts

### `Peliculas.py` — Top Users in Top Movies (MovieLens)

Identifies the **top 10 users** who have watched the most movies from the **top 50 most-watched movies**.

**Steps:**
1. Load all ratings from `u.data` on HDFS.
2. Find the **top 50 most-watched movies** (highest view count).
3. Filter ratings to keep only those for these 50 movies.
4. Count how many of the 50 movies each user has rated.
5. Return the top 10 users by that count.

- **Output:** `User ID → number of top-50 movies watched`

---

### `Script1.py` — Parts Count by Category

Counts how many spare parts belong to each product category.

- **Input:** `hdfs://namenode:9000/user/root/input/REPUESTO.tsv`
- **Key operation:** `groupBy("categoria")` → `count()`
- **Output:** `categoria → cantidadRepuestos` (sorted descending)

---

### `Script2.py` — Available Stock by Warehouse

Calculates the total available inventory units stored in each warehouse.

- **Input:** `hdfs://namenode:9000/user/root/input/INVENTARIO.tsv`
- **Key operation:** `groupBy("id_bodega")` → `sum("stock_disponible")`
- **Output:** `id_bodega → stock_total` (sorted ascending by warehouse id)

---

### `Script3.py` — Total Units Sold per Part

Aggregates the total quantity of each spare part sold across all sales invoices.

- **Input:** `hdfs://namenode:9000/user/root/input/DETALLE_VENTA.tsv`
- **Key operation:** `groupBy("id_repuesto")` → `sum("cantidad")`
- **Output:** `id_repuesto → cantidaTotal` (sorted descending)

---

### `Script4.py` — Purchase Orders by Status

Counts how many purchase orders exist for each order status.

- **Input:** `hdfs://namenode:9000/user/root/input/ORDEN_COMPRA.tsv`
- **Key operation:** `groupBy("estado")` → `count()`
- **Output:** `estado → totalEstado` (sorted descending)

## How to Run

### Generate the dataset

Run the data generator from the Work 1 repository (or a local copy of it):

```bash
cd Tablas   # inside the hadoop-mapreduce-ev-spare-parts repo
python generar_datos.py
```

This produces all required TSV files (~245,000+ rows).

### Upload input files to HDFS

```bash
hdfs dfs -mkdir -p /user/root/input
hdfs dfs -put REPUESTO.tsv      /user/root/input/
hdfs dfs -put INVENTARIO.tsv    /user/root/input/
hdfs dfs -put DETALLE_VENTA.tsv /user/root/input/
hdfs dfs -put ORDEN_COMPRA.tsv  /user/root/input/
hdfs dfs -put u.data            /user/root/input/

# For Peliculas.py — u.item must be on the driver node's local filesystem
cp u.item /tmp/u.item
```

### Submit jobs with spark-submit

```bash
spark-submit Peliculas.py
spark-submit Script1.py
spark-submit Script2.py
spark-submit Script3.py
spark-submit Script4.py
```

### Run on a Docker-based Hadoop + Spark cluster

If your cluster is set up with Docker (e.g., `namenode`, `spark-master` containers):

```bash
docker cp Script1.py spark-master:/tmp/
docker exec spark-master spark-submit /tmp/Script1.py
```
